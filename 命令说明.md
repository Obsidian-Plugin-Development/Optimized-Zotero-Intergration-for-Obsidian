# Optimized Zotero Integration v4.0.0 命令说明

## 核心概念

v4.0.0 把导入拆成了三路独立管道：

```
一篇 Zotero 文献导入后生成的文件：

┌─────────────────────────────────┐
│  ---                            │
│  标题: "Machine Learning..."    │  ← YAML 元数据（属性）
│  作者: ["张三 †", "et al."]     │
│  年份: 2024                     │
│  ---                            │
│                                 │
│  %% Zotero_Notes_Start %%       │
│  ## Abstract                    │  ← 正文（摘要+批注）
│  ...                            │
│  %% Zotero_Notes_End %%         │
│                                 │
│  （用户自己的手写笔记在这里）      │  ← 不会被覆盖
└─────────────────────────────────┘
```

三个管道互不干扰：改 YAML 不影响正文，改正文不影响 YAML。

---

## 一、完整导入类

### 1. 快速导入

**作用：** 从 Zotero 选中文献，一键生成完整的 Obsidian 笔记。

**什么时候用：** 第一次导入文献时。

**例子：** 你在 Zotero 选了一篇论文，运行这个命令 → 自动在你的文献文件夹下生成 `smith2024.md`，包含标题、作者、摘要、笔记等所有内容。

---

### 2. 导入为独立笔记

**作用：** 和快速导入类似，但从 Zotero 的「笔记」入手。

**什么时候用：** 你在 Zotero 里写了笔记，想导出为独立文件到 Obsidian。

**例子：** 你在 Zotero 里对一篇论文写了 3 条笔记，运行此命令 → 3 条笔记分别生成 3 个 md 文件。

---

## 二、插入类（不创建新文件，只往当前文档插入内容）

### 3. 插入条目信息（YAML）

**作用：** 在光标位置插入当前 Zotero 选中文献的 YAML 属性块。

**什么时候用：** 你正在编辑某个 md 文件，想把文献元数据嵌入进去。

**例子：** 光标在文件开头，运行 → 插入：

```yaml
---
标题: "Deep Learning Survey"
作者: ["LeCun Y †", "et al."]
年份: 2015
---
```

---

### 4. 插入批注笔记

**作用：** 在光标位置插入 Zotero 中的笔记和 PDF 批注（高亮、划线等）。

**什么时候用：** 你想把 Zotero 的阅读批注拿到当前笔记中用。

**例子：** 你在写综述，想引用某篇论文的批注 → 运行命令 → Zotero 里那条论文的 PDF 高亮、笔记都插入到光标位置。

---

### 5. 插入引文内容

**作用：** 按 GB/T 7714-2015 国标格式，在光标位置插入完整的格式化引文。

**什么时候用：** 写论文时直接插入一条规范的参考文献引文。

**例子：** 选一篇论文 → 运行 → 插入 `张三, 李四. 机器学习综述[J]. 计算机学报, 2024.`

---

### 6. 插入参考文献

**作用：** 在光标位置插入格式化后的参考文献列表（bibliography）。

**什么时候用：** 写完文章要加参考文献时。

**例子：** 你在 Zotero 选了 3 篇论文，运行命令 → 插入：

```
[1] Smith, J. et al. Machine Learning Advances. Nature, 2024.
[2] Wang, L. et al. Deep Learning Survey. Science, 2023.
...
```

---

### 7. 将笔记插入当前文档

**作用：** 功能与"插入批注笔记"类似（旧版命令）。

**什么时候用：** 用旧版命令，新版推荐用 #4。

---

## 三、增量更新类（v4.0 新特性）

### 8. 更新条目信息

**作用：** **只刷新**已有笔记的 YAML 属性，不动正文。

**什么时候用：** 你在 Zotero 修改了文献信息（比如加了标签、改了影响因子），想把更新同步到 Obsidian，但**不想覆盖手写笔记**。

**例子：** 你之前导入了 50 篇文献，现在 Zotero 里给其中 10 篇加了标签。运行 → 只更新这 10 篇的 YAML 属性，正文和你的手写笔记完好无损。

---

### 9. 更新批注笔记

**作用：** **只刷新** Zotero 笔记和 PDF 批注到已有文件，不动 YAML。

**什么时候用：** 你在 Zotero 里给某篇论文新增了高亮和批注，想同步过来。

**例子：** 你一个月前导入了一篇论文，最近在 Zotero 里又划线了 5 处重点。运行 → 新批注追加到 `%% Zotero_Notes_Start %%` 和 `%% Zotero_Notes_End %%` 之间，YAML 属性和手写笔记不受影响。

---

## 四、辅助类

### 10. 复制引注占位符到剪贴板

**作用：** 在 Zotero 选中文献后运行，引用占位符复制到剪贴板。

**什么时候用：** 写文章时快速引用文献，配合 Pandoc 导出使用。

**例子：** 选中一篇论文 → 运行 → 剪贴板得到 `[@smith2024]`，粘贴到正文里就能在 Pandoc 导出时自动生成参考文献。

---

### 11. 数据浏览器

**作用：** 打开 Zotero 条目数据浏览面板，查看模板变量。

**什么时候用：** 你在写自定义模板时，想看看有哪些字段可用。

**例子：** 打开后选一条文献 → 显示 `title`, `authors`, `date`, `abstract`, `creators[0].firstName` 等所有可用变量 → 帮你写 `{{title}}` 这样的模板。

---

## 引用三兄弟对比

```
插入引文内容     →  张三, 李四. 机器学习综述[J]. 2024.  （完整引文，人读）
复制引注占位符   →  [@zhang2024]                      （标签，机器处理用）
插入参考文献     →  [1] Zhang, L. ...                  （列表，文末放）
```

| 命令 | 输出物 | 写入方式 | 典型用途 |
|------|--------|---------|---------|
| 插入引文内容 | 完整引文句子 | 插入编辑器 | 直接引用 |
| 复制引注占位符 | `[@citekey]` | 复制到剪贴板 | Pandoc 导出 |
| 插入参考文献 | 参考文献列表 | 插入编辑器 | 文末参考文献 |
